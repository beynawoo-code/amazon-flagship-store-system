请求体要点
  • startDate / endDate：YYYY-MM-DD；Insights 最早约 2018-09-01
  • dimension：Insights 必填；ASIN 接口省略 dimension 时仅店级 TOTAL 指标
  • filter：pageIds、sources、tags（AND）；不可同时 sources + tags
  • paginationToken：24h 内、相同请求体可翻页（单次最多约 1500 行）

  • ADS — OpenAPI 定义为 Sponsored Brands 带来的进店流量

Insights：维度 × 指标兼容（摘要）

完整枚举以 OpenAPI InsightDimension 描述为准

dimension ↓ / 指标 → 流量 互动 归因销售 店质量 SQS
DATE VISITS, VIEWS, VISITORS* DWELL_TIME, BOUNCE_RATE, NEW_TO_STORE ORDERS, UNITS, SALES SCORE_LEVEL, RECOMMENDATIONS, …
PAGE VISITS, VIEWS DWELL_TIME, BOUNCE_RATE ORDERS, UNITS, SALES —
SOURCE VISITS, VIEWS DWELL_TIME, BOUNCE_RATE ORDERS, UNITS, SALES —
TAG VISITS, VIEWS DWELL_TIME, BOUNCE_RATE ORDERS, UNITS, SALES —
STORE 店级汇总 DWELL_TIME, BOUNCE_RATE, NEW_TO_STORE — —

* VISITORS 仅支持 DATE；不受 filter 影响。STORE 维度或省略 dimension 可做店级 DWELL_TIME / BOUNCE_RATE。

Brand Store Analytics API

读数：店流量、页面、来源、Tag、ASIN 漏斗、店质量建议、视频播放。

✅ 适合：多站点 KPI、改版前后、SB 落地店对照（SOURCE=ADS）

Store Builder / Console

搭店、模块、素材、审核发布；Store Quality 操作建议的 CTA 链回 Builder。

❌ Analytics API 不替代建店与过审流程

Attribution API + Tag

站外活动、Publisher、click-through URL（可指向 Brand Store）。

🔗 Store Insights dimension=TAG 消费 Tag 维度结果，不创建 Tag

Sponsored Brands

落地 Brand Store 的广告流量；与 Store SOURCE=ADS 最可对齐。

⚠️ 全账户 SP/SD 花费 ≠ 全店 visits

SP-API / 财务 BI

订单、库存、Buy Box、类目份额。

❌ 解释「有点击无订单」需并行接入

你们 ERP 子系统（P0）

统一日/站点/页/ASIN 维度；模块分类、场景映射、版本档案为自建维表。

§2 标准路径：步骤清单

Phase A ERP 内（必须先做）

步操作产出
A1进入 Tag 中心 → 创建向导：品牌 → 站点 → SKU → Amazon Channel（5 选 1） → 营销大类 → 营销子类 → Publisher →（红人）是否付费 → 活动日期amazon_channel + strategy_* + publisher_name
A2命名预览与校验：无冲突提交；有冲突改日期/渠道或 SO 确认 SKUtag_request_id，状态 pending_amazon
A3记录建议 Campaign 名、Ad Group 名、落地页类型（PDP / Store）供 Phase B 照抄
强约束 K1：canonical_name = {品牌}-{站点}-{SKU}-{name_short}-{YYYYMMDD}。name_short = 营销大类+子类（例 PSoINF，勿仅用 INF）。见 渠道字典。

Phase B Amazon Console（官方 SOP 对齐）

步操作Amazon 字段
B1登录 Advertising Console → Measurement & Reporting → Amazon Attribution → Create campaign—
B2Creation method 选 Create manuallycreation_method
B3Products：推广 SKU + Add variations；缺货/无图/无价不可选measured_asins[]
B4Ad group name：粘贴 ERP canonical_namead_group_name
B5Publisher：见 Publisher 选型 — Google 选 google ads；FB/IG/联盟/KOL 用 Newpublisher
B6Channel：与 ERP amazon_channel 一致（Search / Social / Video / Email / Display）amazon_channel
B7Click-through URL：单品 → PDP；多品/品牌 → Brand Storeclick_through_url
B8保存后复制 Attribution Tagattribution_tag_raw
B9Tag 填入站外广告最终 URL—

Phase C 回写 ERP

步操作系统字段
C1Tag 列表 → 链接与 Attribution 同步；粘贴 Tag 或填 Campaign/Ad Group IDattribution_tag_raw, amazon_campaign_id, amazon_ad_group_id
C2状态改为 active；无点击可按规范归档status
C3（可选）Attribution API 定时拉报表或 Console 核对首周last_report_sync_at

§4 字段字典（Amazon Console）

字段必填说明
Campaign name是建议 {品牌}_{站点}_{主题}_{YYYYMM}
Creation method是manual | bulk_upload
Products (ASIN)是在库、有图有价；同品牌非推广品计入 Total
Ad group name是必须与 ERP canonical_name 一致
Publisher / Channel是长尾渠道 Publisher 选 New
Click-through URL是PDP 或 Store URL
Attribution Tag生成挂站外 URL；Google/Meta 可用 API macro 模板

§7 验收检查表

  • ERP 已有 tag_request_id；amazon_channel + strategy_major_code + strategy_minor_code 合法
  • name_short 为组合短码（营销大类+子类），已通过唯一性校验
  • Amazon Ad group name 与 canonical_name 完全一致
  • ASIN 在库、有图有价；变体已 Add variations
  • Channel / Publisher 与站外投放一致
  • Click-through URL 正确（PDP vs Store）
  • attribution_tag_raw 已回写，状态 active
  • 站外 URL 已挂 Tag，抽查点击正常
  • （US）BRB 已 enrollment 且报表含 brb_bonus
